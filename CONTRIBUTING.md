# 🤝 Contributing to LuusOS & Credits

Welcome! We are thrilled that you want to help make **LuusOS** better. Whether you are fixing a bug, updating documentation, adding drivers, or testing the OS on new environments (like Windows, Termux, or Limbo), your support is highly appreciated!

---

## 🇧🇷 Diretrizes de Contribuição (Português)

### 🚀 Como posso ajudar?
1. **Reportando Bugs:** Abra uma *Issue* detalhando o erro, a arquitetura alvo (x86 ou AArch64) e o emulador utilizado (QEMU, VirtualBox, Limbo).
2. **Sugerindo Melhorias:** Se você tem ideias para novos drivers VGA, gerenciamento de memória ou novas rotinas em Assembly/Rust, abra uma discussão nas *Issues*.
3. **Enviando Código (Pull Requests):**
   * Faça um *Fork* do repositório.
   * Crie uma branch para sua modificação (`git checkout -b minha-nova-feature`).
   * Garanta que o código compila perfeitamente rodando `make all`.
   * Envie o *Pull Request* explicando suas alterações.

## 🇺🇸 Contribution Guidelines (English)

### 🚀 How Can I Help?
1. **Reporting Bugs:** Open an *Issue* describing the bug, your target architecture (x86 or AArch64), and the environment used (QEMU, VirtualBox, Limbo PC Emulator).
2. **Feature Requests:** Have an idea for a new text-mode feature, memory paging, or specific C/Rust optimizations? Let us know in the issues section!
3. **Submitting Code (Pull Requests):**
   * *Fork* this repository.
   * Create your feature branch (`git checkout -b feature/amazing-driver`).
   * Make sure your build changes compile clean via `make all`.
   * Open a *Pull Request* documenting your implementations.
